# 获取浏览器网址

部署完成后，你会得到一个类似这样的地址，为网络安全，

1.小范围长期信任接入：建议使用基于 WireGuard 协议的 VPN（ZeroTier/Easytier/Talescale）或'蒲公英'进行“虚拟局域网”，即“异地组网”访问。
2.不特定对象、公网分享：建议使用基于 FRP 网络协议的 内网穿透 软件（节点小宝/贝锐花生壳）进行“端口转发”到公网IP。您需要拥有全球唯一单播地址。

```
https://xxxx.streamlit.app
```

---

## 第一步：打开 Streamlit 部署网站

浏览器访问：**https://share.streamlit.io**

点击 **Sign in**（或 **Continue with GitHub**），用 GitHub 账号*keil510086-commits* 登录并授权。

---

## 第二步：新建应用

1. 登录后点击 `Create app`（或 `New app`）
2. 按下面填写：

| 项目 | 填写 |
|------|--------|
| Repository | keil510086-commits/- |
| Branch | main |
| Main file path | app.py |
| App URL（如有） | 可填`banhui`（ASCII，作为网址的一部分） |

3. 点击 `Deploy`

等待 1～3 分钟，出现 **Your app is live!** 即成功。

---

## 第三步：复制网址

页面上会显示你的应用地址，例如：

```
https://banhui.streamlit.app
```

把这个链接保存下来，班会时直接打开即可。

---

## 第四步：使用前填接口密钥

打开网址后：

1. 点击页面**左上角 `>`** 展开侧边栏
2. 填写 **接口密钥**、**接口地址**、**模型名称**（务必准确输入，如：Deepseek-v4-pro）
3. 自动出现 GM 开场白，开始互动

> 建议班会开始前填好密钥，然后收起侧边栏，避免投影时露出。

## 常见问题

**Q：部署失败？**  
先确认 GitHub 上能看到 `app.py` 和 `requirements.txt`。

**Q：仓库名如何修改？**  
可在 GitHub 仓库 **Settings → Repository name** 改名为 `banhui`，然后在 Streamlit 重新选仓库部署。

**Q：网址要收费吗？**  
Streamlit Community Cloud 免费，公开仓库即可部署。
